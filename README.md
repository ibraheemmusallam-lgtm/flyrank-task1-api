# Task API

A small CRUD API for managing a to-do list, built with Python and FastAPI. Tasks are stored in memory and can be created, listed, retrieved, updated, and deleted through HTTP requests or the interactive Swagger UI.

## Features

- Full CRUD operations for tasks
- In-memory task storage with three example tasks
- Input validation with JSON error responses
- Appropriate HTTP status codes
- Interactive Swagger UI generated by FastAPI
- Health endpoint for checking whether the service is running

## Requirements

- Python 3.10 or newer
- Git Bash on Windows, or another terminal with Python available

## Installation

From the project directory in Git Bash:

```bash
py -3.11 -m venv .venv
source .venv/Scripts/activate
```

Install FastAPI and its standard dependencies, including the Uvicorn server:

```bash
python -m pip install -r requirements.txt
```

The `requirements.txt` file installs `fastapi[standard]`.

## Running the API

Start the development server with one command:

```bash
fastapi dev main.py
```

The API will be available at <http://localhost:8000>.

Interactive Swagger documentation is available at <http://localhost:8000/docs>.

## Task structure

Each task contains:

```json
{
  "id": 1,
  "title": "Learn FastAPI",
  "done": false
}
```

The server generates `id`, and every newly created task starts with `done` set to `false`.

## Endpoints

| Method | Path | Description | Success | Errors |
|---|---|---|---:|---:|
| `GET` | `/` | Return API metadata | `200` | - |
| `GET` | `/health` | Confirm that the server is running | `200` | - |
| `GET` | `/tasks` | Return all tasks | `200` | - |
| `GET` | `/tasks/{task_id}` | Return one task by ID | `200` | `404` |
| `POST` | `/tasks` | Create a task from a non-empty title | `201` | `400` |
| `PUT` | `/tasks/{task_id}` | Update a task's title and/or completion status | `200` | `400`, `404` |
| `DELETE` | `/tasks/{task_id}` | Delete a task with an empty response body | `204` | `404` |

## Example requests

Create a task:

```bash
curl -i -X POST http://localhost:8000/tasks -H "Content-Type: application/json" -d '{"title":"Buy milk"}'
```

Update task 4:

```bash
curl -i -X PUT http://localhost:8000/tasks/4 -H "Content-Type: application/json" -d '{"title":"Buy oat milk","done":true}'
```

Delete task 4:

```bash
curl -i -X DELETE http://localhost:8000/tasks/4
```

## Error responses

Unknown task IDs return `404 Not Found`:

```json
{
  "error": "Task 99 not found"
}
```

Missing, empty, or invalid task fields return `400 Bad Request` with a JSON error message.

## In-memory storage

This project intentionally does not use a database or files for task storage. Changes remain available only while the server process is running. Restarting the server restores the three example tasks from `main.py`.
